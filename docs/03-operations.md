# C. 運用フロー・管理システム

WordPress管理画面ベースでの運用体系を定義します。

## C-1. 役割分担

| 役割 | 担当 | 主な作業 |
|---|---|---|
| **システム保守** | 司令官 | テーマ/プラグイン更新、トラブル対応、バックアップ監視、CPT/ACF変更 |
| **コンテンツ運用** | KARINTOW側スタッフ | イベント追加・編集、バナー差替、モデル登録、お知らせ投稿 |
| **デザイン制作** | KARINTOW側スタッフ + Canva | バナー画像制作（テンプレート使用） |
| **現場運営** | 現場スタッフ | 接客・撮影業務（サイト連動なし） |

### 前提

- KARINTOW側スタッフは **WordPress管理画面の操作経験あり**（旧サイト運用で慣れている）
- SWELLのブロックエディタは直感的、学習コスト低
- 操作ミス防止のため、権限を編集者以下に制限

## C-2. 日常運用の典型タスク

### タスク1: 新規イベント追加

頻度: 週数回

**手順:**
1. KARINTOW担当者がCanvaでバナー画像制作
2. WordPress管理画面 → `イベント` → `新規追加`
3. タイトル入力（イベント名）
4. 本文エディタ（SWELLブロック）で説明文を入力
5. アイキャッチ画像にバナーをアップロード
6. ACFフィールドを入力:
   - 開催日・時刻
   - 会場（プルダウンから選択）
   - 出演モデル（リレーション検索で複数選択）
   - 料金
   - ステータス: `予約受付中`
7. カテゴリにチェック（セッション / SP撮影会 / 個撮 / キカク / スペシャル）
8. `公開` ボタン

**所要時間目安**: 1イベント10分

### タスク2: イベント情報の修正

1. `イベント` 一覧 → 該当をクリック
2. 修正 → `更新`

### タスク3: 満員/終了処理

1. 該当イベント編集画面
2. ACFの `ステータス` を `満員` または `終了` に変更
3. `更新`

→ テンプレート側で条件分岐し、一覧・詳細に「満員」バッジを自動表示

### タスク4: 新規モデル登録

1. 写真素材を受領
2. `モデル` → `新規追加`
3. タイトル（モデル名）、本文（自己紹介）
4. アイキャッチ画像（プロフィール写真）
5. ACFフィールド（丸サムネ画像、フリガナ、事務所、SNS等）
6. `現役フラグ` を ON
7. `公開`

### タスク5: モデル引退・非表示化

1. 該当モデル編集画面
2. ACFの `現役フラグ` を OFF
3. `更新`

→ 一覧・詳細ページから即座に消える。過去のイベント出演履歴は残る。

### タスク6: TOPヒーロー画像の差替

頻度: 月1〜2回

**手順（SWELL使用時）:**
1. `外観` → `カスタマイズ` → `トップページ` → `メインビジュアル`
2. 画像を差替え
3. `公開`

※ KARINTOW担当者にも許可（SWELLのカスタマイザーは編集者権限でも使える設定が可能）

### タスク7: お知らせ投稿

1. `投稿` → `新規追加`
2. タイトル・本文入力
3. カテゴリに「お知らせ」選択
4. `公開`

→ TOPの「最新のお知らせ」に自動で最新3件表示

## C-3. 権限設計

### WordPressユーザーロール

| ロール | 担当者 | できること |
|---|---|---|
| **Administrator** | 司令官 | 全権限 |
| **Editor** | KARINTOW担当者 | 投稿・固定ページ・CPT編集、公開、画像アップロード。プラグイン/テーマ/ユーザー管理は不可 |

### 権限制限プラグイン（推奨）

- `User Role Editor`: ロールごとの権限を細かく調整
- Editorに対して以下は明示的に許可：
  - CPT `event`, `model`, `agency`, `venue` の編集
  - ACFフィールドの編集
  - カテゴリ（タクソノミー）の追加・編集
  - SWELLカスタマイザーのメインビジュアル部分
- 以下は明示的に禁止：
  - プラグイン有効化/無効化
  - テーマ変更
  - WordPress本体更新

## C-4. Canvaテンプレート運用

### テンプレート構成

司令官側で作成しKARINTOW共有フォルダへ納品:

| テンプレート | サイズ | 用途 |
|---|---|---|
| イベントバナー横長 | 1200×630 | イベントのアイキャッチ画像 |
| イベントカード正方形 | 1080×1080 | SNSシェア用 |
| ヒーロー背景 | 1920×1080 | TOPメインビジュアル |
| カテゴリカード背景 | 600×600 | カテゴリブロック背景 |
| モデル紹介カード | 1080×1350 | SNS用 |

### 運用ルール

- ブランドカラー・フォントはテンプレートで固定
- ロゴは支給データから変更不可
- Canvaブランドキット機能での強制化を検証（Q25）

## C-5. バックアップ・復旧

### 自動バックアップ（ConoHa WING標準）

- **自動バックアップ14日分** が標準で無料提供
- データベース + Webコンテンツ領域両方対象
- 復元は管理画面から1クリック

→ 前回の「rootパスワード喪失で復旧不可」事故は**構造的に起きない**

### 追加バックアップ（プラグイン）

- `UpdraftPlus` 無料版で週次バックアップ
- 保存先: Google Drive等の外部ストレージ
- ConoHa標準バックアップと二重化

### 手動バックアップ

司令官が月1回実施:
- `ツール` → `エクスポート` でWXR形式の全データダンプ
- GitHubリポジトリ `karintow-gsta-legacy/backups/` にコミット
- アップロード画像は別途zipで保管

### 復旧手順

**ケース1: プラグイン更新等で画面が壊れた**
→ ConoHa WING管理画面 → 自動バックアップから該当日を選択 → 復元

**ケース2: データ破損・改ざん**
→ 同上 + UpdraftPlusのGoogle Driveバックアップも選択肢に

## C-6. 障害対応・問い合わせ窓口

### 一次対応

- KARINTOW担当者がサイト不具合に気付いた場合 → 司令官へLINE/メール連絡
- 緊急度高（完全停止）→ 即時連絡
- 緊急度低（表示崩れ等）→ 業務時間内

### 司令官の対応

1. ConoHa WING管理画面で稼働状態確認
2. WP管理画面にログインできるか確認
3. 必要ならConoHa自動バックアップから復元
4. エラーログ確認（`wp-content/debug.log`）

## C-7. SEO・分析

### 必須設定

| 項目 | ツール |
|---|---|
| サイトマップXML | プラグイン `XML Sitemap Generator` or SEO SIMPLE PACK標準機能 |
| SEOメタタグ | `SEO SIMPLE PACK`（SWELL開発者製、相性◎） |
| Google Search Console登録 | 手動 |
| Google Analytics 4設置 | SEO SIMPLE PACK or SWELL標準 |
| OGP / Twitter Card | SEO SIMPLE PACK |
| robots.txt | WP標準 + プラグインで編集可 |

### 月次レポート

司令官側で月1回（❓ Q23: 要否確認）:
- 訪問者数推移（GA4）
- 人気ページTOP10
- 流入元
