# A. サイトマップ・ページ構成

## A-1. 全体ナビゲーション

新サイト（WordPress + SWELL）のグローバルナビゲーション構成：

```
┌─ Gスタ撮影会 ロゴ
│
├─ TOP                      /
├─ イベント                 /event/
│   ├─ セッション           /event/category/session/
│   ├─ SP撮影会             /event/category/sp/
│   ├─ 個撮                 /event/category/one/
│   ├─ キカク               /event/category/kikaku/
│   └─ スペシャル           /event/category/special/
│
├─ モデル一覧               /model/
│   └─ モデル詳細           /model/{slug}/
│
├─ 初めての方へ             /beginners/
├─ アクセス                 /access/
├─ お申し込み               /apply/
├─ お問い合わせ             /contact/
└─ モデル募集               /audition/
```

### URL設計方針

- イベントは WP カスタム投稿タイプ `event` のスラッグを使用
- カテゴリは WP カスタムタクソノミー `event_category`
- モデルは WP カスタム投稿タイプ `model`
- 旧URL `/event/27397/` は **そのまま継承可能**（数値スラッグで保持）
  → 旧サイトからのSEO・SNSシェアの資産を温存

## A-2. 現行サイトとの対応関係

現行サイト（613ページ）の構造を新サイトに集約：

| 現行 | 件数 | 新サイトでの扱い |
|---|---|---|
| `/event/{ID}/` | 約150件 | カスタム投稿タイプ `event` として移行（同URL継承） |
| `/one/{ID}/` | 約120件 | `event` に統合（カテゴリ=個撮）+ `/one/{ID}/` 301リダイレクト |
| 固定ページ19件 | 19 | 一部廃止、現役のものだけ移植 |
| `/page/2~6/` | 5 | WP標準のページネーション |
| `/feed/` | — | WP標準RSSで代替 |

## A-3. TOPページ構成（フレッシュ撮影会インスパイア）

上から順に：

```
1. ヒーロー
   - 背景: 白 + SVG装飾オブジェクト散布
   - 英字キャッチ "Fresh cuteness always nearby" 的な
   - 日本語キャッチ
   - バナースライダー（横長、旧Gスタのトップページ管理機能継承）
   - [← →] ナビ、ドットインジケータ

2. Tomorrow's Event / 明日のイベント
   - 英字ラベル + 日本語見出し
   - 日付切替（前日/翌日）
   - 開催イベントごとに「カテゴリタグ + タイトル + 参加モデル丸サムネグリッド」
   - フレッシュ方式の直接参考

3. Event Schedule / 今月のイベント
   - 月縦スクロール型カレンダー
   - 各日付に該当イベントのバナーを縦に並べる
   - 前月/次月切替
   - ⭐ プラグイン不要、ACF日付フィールド + 自作テンプレートで実装

4. カテゴリナビ（5種）
   - SVGアイコン形式（セッション/SP撮影会/個撮/キカク/スペシャル）
   - 白背景カード、赤アクセント、ホバーで浮遊

5. Fresh model / Gスタモデル一覧
   - 円形サムネイルグリッド（旧Gスタ資産継承）
   - 12〜20名
   - 「View All」ボタンで /model/ へ

6. モデル募集バナー
   - 大判ビジュアル + キャッチコピー
   - → /audition/

7. News / お知らせ            ❓ Q14 要確認（フレッシュには存在、Gスタは要判断）
   - 最新5件

8. Concept / コンセプト
   - Gスタ撮影会のブランドストーリー
   - 背景画像 + テキスト

9. 初めての方・バナー群
   - オンラインストア、はじめての方、バイト募集等
   - フレッシュ方式

10. アクセス
   - Google Map + 住所・最寄駅

11. フッター
   - SNS、サイトマップ、コピーライト
```

詳細なデザイン仕様は [06-design-direction.md](./06-design-direction.md) を参照。

## A-4. ページごとの主要要素

### TOPページ
詳細は A-3 参照。

### イベントカテゴリページ（`/event/category/{slug}/`）
- WP カテゴリアーカイブテンプレート（`taxonomy-event_category.php`）
- カテゴリ説明文（タクソノミー説明）
- 並び順: ACF日付フィールド近い順

### イベント詳細ページ（`/event/{slug}/`）
- カスタム投稿タイプ `event` シングルテンプレート（`single-event.php`）
- ACFフィールド: メインバナー、日付、時刻、会場、料金、出演モデル（リレーション）
- 申し込みボタン

### モデル一覧（`/model/`）
- カスタム投稿タイプ `model` のアーカイブ（`archive-model.php`）
- 50音順 or 事務所別フィルタ
- 丸サムネイルグリッド

### モデル詳細（`/model/{slug}/`）
- `single-model.php`
- ACFフィールド: プロフィール写真、所属事務所、SNS、Bio
- 過去出演イベント一覧（リレーションの逆引き）

### 初めての方（`/beginners/`）
- 固定ページ + SWELLブロックエディタで構築
- ステップ別ガイド・FAQ

### アクセス（`/access/`）
- 固定ページ
- Google Map 埋め込み・住所・営業時間

## A-5. URL設計方針

### ✅ 旧URL継承

WordPressへの移行なら、旧URL `/event/27397/` を**そのまま使えます**。
カスタム投稿タイプのスラッグを数値ID名で登録すれば良いだけ。

→ 301リダイレクトの大量設定が不要、SEO・SNSシェア資産を完全保全。

`/one/{ID}/` のみ、新カテゴリ「個撮」配下の `/event/{ID}/` に統一するため、
301リダイレクトを設定（`.htaccess` で正規表現1行）。

## A-6. レスポンシブ対応方針

- モバイルファースト（SWELL標準）

  - 本文最小16px（SWELLカスタマイザーで設定）
  - ボタン高さ最小44px
  - 行間1.7以上
