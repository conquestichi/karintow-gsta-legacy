## D-2. 技術スタック

### 採用技術

| レイヤー | 技術 | 理由 |
|---|---|---|
| ホスティング | **Webflow Hosting** | CDN込み、保守不要、スケーラブル |
| CMS | **Webflow CMS** | ノーコードで非エンジニアが運用可能 |
| デザインツール | **Webflow Designer** | デザインとマークアップを統合管理 |
| 画像制作 | **Canva** | 既にKARINTOW社内で利用中 |
| ドキュメント管理 | **GitHub (Private)** | バージョン管理・透明性 |
| ドメイン | 既存のレジストラ | 既存契約をそのまま使用予定 |

### 採用しなかった技術と理由

| 技術 | 理由 |
|---|---|
| WordPress 維持 | 現行WPは古すぎ、保守コスト高、root権限不明、開発会社不通 |
| WordPress + 新テーマ | 結局VPS保守が必要、CentOS 7はEOL、リスク高 |
| 自前サーバー新調 | 司令官の保守負担が大きい、Webflow代替で十分 |
| Wix / Studio | Webflowの方がCMS設計の自由度が高い |
| Headless CMS + Next.js | 過剰、KARINTOW担当者が運用できない |

## D-5. 移行手順（マスタープラン）

### Phase 1: 設計確定 ← 現在ここ

- ✅ 旧サイト情報収集
- ✅ wgetミラー
- ✅ デザイン方針確定
- 🚧 設計ドキュメント整備（本ドキュメント）
- ❓ ブランドカラー最終確定

### Phase 2: 新サイト本格構築

- CMSコレクション設計を Webflow に実装
- 全セクションをデザイン仕様通りに構築
- レスポンシブ対応
- ロゴ正式実装
- アニメーション・ホバー仕上げ

### Phase 3: データ移行

1. **旧HTMLパース**
   - GitHubリポジトリの `event/*/index.html` 約150件をパース
   - 抽出: タイトル、日付、カテゴリ、料金、説明、出演モデル、バナーURL
   - CSV化（イベント.csv、モデル.csv）
2. **マスタデータ整備**
   - Categories: 手動5件登録
   - Venues: 手動登録（数件）
   - Agencies: CSV準備
   - Models: CSV準備
3. **画像移行**
   - 必要な画像のみ選別（古いキャンペーンは除外）
   - 肖像権チェック → 各モデルから許諾再取得
   - Webflow Asset Manager にアップロード
4. **CMSインポート**
   - マスタ系（Categories, Venues, Agencies, Models）から先にインポート
   - 後にイベントをインポート（リレーションを正しく結ぶ）
5. **検証**
   - リンク切れチェック
   - 画像表示チェック
   - レスポンシブチェック

### Phase 4: 公開準備

- メタタグ・OGP最終調整
- Google Search Console / GA4 設定
- robots.txt / sitemap.xml 設定
- 301リダイレクト準備（旧URL → 新URL）
- KARINTOW担当者向けレビュー

### Phase 5: ドメイン切替（カットオーバー）

1. Webflowにカスタムドメイン `studio-g.net` を追加
2. DNS設定変更（さくらVPS → Webflow CNAMEへ）
3. SSL証明書発行確認
4. 切替後動作確認
5. 301リダイレクト動作確認

### Phase 6: 並行運用 → 旧サイト停止

- 1〜2ヶ月並行運用
- 問題なければ旧VPS解約
- KARINTOW社内向け運用マニュアル納品
